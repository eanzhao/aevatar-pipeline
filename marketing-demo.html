Marketing Campaign Orchestrator Demo

这页按阶段展示同一条 workflow 的 graph 和原始 YAML，方便对照每一段实际在做什么。

Steps 95

从接收需求到导出结果，全部在一条 run 内完成。

Roles 7

解析、找源、研究、策略、交付、导出各司其职。

`marketing_search_mcp`、`marketing_extract_mcp`、`marketing_markdown_export`。

Overview

这条 workflow 的作用

输入

一段自由输入的营销研究需求，可能包含品牌名、官网、目标渠道和交付预期。

中间过程

LLM 负责解析、归纳和报告生成；connectors 负责搜索、页面抽取和文件导出；必要时走 `human_input` 回补品牌资料。

输出

一份 evidence-grounded markdown discovery report，加上一段导出结果元数据。

Phase 1

接收需求与品牌源定位

这一段先把原始需求整理成结构化 brief，再判断能不能直接拿到官网。如果拿不到，就走搜索；如果搜索还不稳，就进入人工补充分支。
